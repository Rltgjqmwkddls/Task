DLCR: 옷을 갈아입는 사람(RE-ID)를 위한 확산을 통한 생산적 데이터 확장 프레임워크

RE-ID의 주요 목표는 서로 다른 시간과 장소에서 촬영된 이미지나 영상에서 동일한 사람을 매칭하는 것
최근 CC-REID 연구들은 구조적(옷과 무관한) 정보와 외형적(옷과 관련된) 정보를 분리
기존 재식별 모델 아키텍처와 손실 함수에 변별적인 수정을 가하는 경우가 많아졌다.
그러나 이 방식은 데이터셋에서 옷의 변화가 적다는 한계로 제약을 받고 있음
이 문제를 해결하기 위해 데이터 확장 프레임워크(DLCR)를 제안

데이터 생성과 재식별 훈련 및 예측 정교화 두 단계로 구성

첫 번째 단계
추가 샘플을 생성하여 주어진 CC-ReID 훈련 데이터셋을 확장이 목표
(현실적인 이미지를 생성, 편집이 가능한 Diffusion 모델 사용)
Diffusion만 사용하면 어려움이 있어 바이너리 마스크를 생성 -> 인간 파싱 방법을 활용하여 해결
의류 항목의 텍스트 설명이 없어, LLaVA와 LLaMA를 함께 활용
두 번째 단계


첫 단계에서 생성된 데이터를 사용하여 모델을 훈련하고 옷이 바뀌는 상황에서 어떤 성능 향상이 있는지 보여줌

첫 번째 단계에서 LLaVA와 LLaMA, Diffusion을 활용하여 이미지 내 주체의 옷을 제어 가능하게 변경
두 번째 단계에서는 생성된 데이터를 활용하여 점진적 학습 전략과 함께 10배 많은 데이터를 학습시키되, 훈련 시간에 거의 영향을 미치지 않는다.

1단계. 데이터 생성
첫 번째 단계에서는 피사체의 옷 부분을 인페이팅하여 훈련 이미지의 다양한 변형을 생성
(이를 위해 텍스트 기반 모델인 Stable Diffusion Inpainting)을 사용해 옷 부분을 설명, 인간 파싱을 통해 생성된 의미적 마스크로 의상 영역을 지정)
특정 피사체의 다양한 이미지를 생성할 때, 인물 관련 정보를 보존하는 것이 중요
먼저, LLaVA에 프롬프트를 사용하여 각 이미지에서 피사체가 착용한 상의, 하의, 신발에 대한 정보를 추출
(그러나 특정 의상에 대한 이미지들은 가림, 조명, 시점 변화로 인해 피사체의 가시성이 달라질 수 있으며, 일부 이미지가 누락되거나 잘못된 의류 항목을 응답하는 경우가 발생할 수 있음
이미지 기반 응답을 LLaMA에 입력하여 이를 요약하도록하고 종합적인 의류 설명을 얻는다.

2단계. 재식별 훈련 및 정교화
두 번째 단계에서는 예측 정교화 전략을 제안하며, Diffusion 기반의 인페이팅 기법을 독창적인 테스트 시점 증강으로 사용하여, 평가 시 보다 전체적이고 앙상블 기반의 의사 결정을 제공
훈련 시간에 미치는 영향을 최소화하기 위해 쌍방 방식(데이터를 점진적으로 훈련 배치에 추가)을 사용
각 원본 훈련 이미지에 해당하는 생성 샘플 중 하나를 쌍으로 만들어 배치 크기를 두 배로 늘립니다.
옷이 변결된 생성 이미지를 점진적으로 도입하는 방식이 더 나은 결과를 가져온다는 것을 확인
(10배 많은 데이터를 생성했기 때문에, 점진적 학습 전략을 통해 모델이 생성 데이터를 효율적으로 활용하면서도 훈련 시간에 큰 영향을 주지 않도록 함

훈련 초기 단계에서 노출되는 옷 변경 이미지를 점진적으로 늘려가며, 모델이 옷차림에 무관한 특징을 학습하는 난이도를 점진적으로 증가시킵니다. 모델은 훈련이 진행될수록 피사체의 다양한 의상에 대해 점차 무시하도록 학습하여 옷차림에 구애받지 않는 특징을 효과적으로 학습하게 됨

본 실험에서는 기본 CC-ReID모델을 사용 모든 데이터셋에서 생성된 데이터를 활용했을 때 가장 우수한 성능을 보임
최근 데이터셋과 비교를 했을 때 성능 향상이 된거 같음

결론은 DLCR이 CC-ReID 성능을 크게 향상시키며, 여러 벤치마크 데이터셋에서 SOTA 결과를 달성함을 확인 

DLCR 논문을 본 소감
일단은 옷을 갈아입는 사람을 위한 거 같은데 모든 사람들이 옷을 갈아입고 생활을 함
그리고 비슷한 옷 똑같은 옷을 계속 입는다고 해도 날마다 차이가 있음
이게 대체 어떤 목적을 가지고 있는지 모르겠음
얼굴인식이 아닌데 동일한 의류를 매칭한다? 이게 무슨 의미가 있나 생각이 들었음
사람이 많은 곳에서는 잘 인식이 되지 않을거 같고 카메라 각도에 따라 인식이 잘 될 수도 안될 수도 있는거 같음
